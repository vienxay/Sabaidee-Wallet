📋 ຄຳອະທິບາຍເສັ້ນເຊື່ອມຕໍ່

Composition (ເປັນເຈົ້າຂອງ - ລຶບນຳກັນ)
Association (ມີຄວາມສຳພັນ)
Dependency (ກວດສອບ/ເພິ່ງພາ)
1 = ໜຶ່ງ
1..* = ໜຶ່ງ ຫຼື ຫຼາຍ
0..* = ສູນ ຫຼື ຫຼາຍ

📝 ສະຫຼຸບຄວາມສຳພັນ

ຈາກ Class ປະເພດ ໄປຫາ Class Multiplicity ຄວາມໝາຍ
User ◆ Composition Wallet 1 → 1..* User 1 ຄົນ ມີ Wallet 1 ຫຼື ຫຼາຍອັນ
Wallet ◆ Composition Transaction 1 → 0..* Wallet 1 ອັນ ມີ Transaction 0 ຫຼື ຫຼາຍລາຍການ
Merchant ─ Association Transaction 1 → 0..* Merchant 1 ຮ້ານ ສ້າງ Transaction 0 ຫຼື ຫຼາຍລາຍການ
Admin - - → Dependency User — Admin ກວດສອບ User
Admin - - → Dependency Wallet — Admin ກວດສອບ Wallet
Admin - - → Dependency Transaction — Admin ກວດສອບ Transaction
Admin - - → Dependency Merchant — Admin ກວດສອບ Merchant
